# 📘 부기맨(Boogieman) 도서 관리 시스템 요구사항 정의서

## 1. 프로젝트 개요

**부기맨(Boogieman)**은 사내 구성원이 원하는 희망 도서를 편리하게 신청하고, 회사가 보유한 도서를 쉽게 검색하여 대출/반납할 수 있도록 지원하는 사내 도서 복지 관리 시스템입니다.

**최소 기능 제품(MVP) 목표:** 사원들이 알라딘 API를 통해 정확한 도서 정보(ISBN13, 고해상도 표지, 분야)로 희망 도서를 신청하고, 실물 도서를 대출/반납하며, 보유 도서를 검색하는 "핵심 도서 순환 사이클"이 우선 오동작 없이 운영되는 것을 MVP 목표로 합니다. (R1 기능 중심)

---

## 2. 목표 사용자

* **일반 사원 (User)**: 외부 검색(알라딘 연동)을 통한 빠르고 정확한 희망 도서 신청, 도서 검색 및 대출/반납 기능, 마이페이지 대출/예약 현황 조회 기능 사용
* **관리자 (Admin)**: 알라딘 검색 연동을 통한 손쉬운 신규 도서 입고(등록, ISBN13 기준), 신청 도서 승인/거절, 도서 상태 현황 관리, 연체 및 예약 취소/승계 관리

---

## 3. 기술 스택

| Concern | Technology |
|---------|-----------|
| Database | MariaDB 11.3 |
| Cache | Redis 7 |
| Migrations | Flyway (`src/main/resources/db/migration/`) |
| Auth | JWT (JJWT 0.11.5) + Kakao OAuth |
| External API | **Spring RestClient (알라딘 ItemSearch API 연동 — ISBN13 정규화, 고해상도 표지)** |
| Complex queries | QueryDSL |
| API docs | SpringDoc OpenAPI (Swagger UI at `/swagger-ui.html`) |
| Templates | Thymeleaf |
| Frontend | Vanilla JS + Tailwind CSS |
| Notifications | Redis Pub/Sub + SSE (Server-Sent Events) |

## 4. 제약 조건 (보안/성능/비즈니스 룰)

### 🔒 보안 (Security & Safety)

1. **IDOR (접근 제어 취약점) 방지 철저**: 예약 취소 통신이나 도서 반납 기능의 엔드포인트에서 타인의 `borrowId`, `reservationId`를 파라미터로 무단 지정하여 호출하는 악의적 조작이 불가능하도록, 백엔드에서 인증 토큰(세션 권한) 소유 여부를 기반하는 검증 과정 (403 Forbidden 등으로 블록)이 필수적입니다.

2. **CSRF, XSS 방어**: 도서 신청 사유 등 사용자의 텍스트 입력 구간에 악성 스크립트 작성 시도를 막기 위한 XSS 필터링 적용.

3. **Hard Delete 사용 금지 (Soft Delete 전환 제안)**: 회원이 부서를 이동/탈퇴하거나 도서가 분실/폐기되었을 때 테이블 레코드를 삭제(`DELETE`)하는 것이 아닌 `is_deleted` 컬럼이나 `status="REMOVED"`로 설계하여 시스템 안의 과거 "예전 통계 및 이력"이 깨지는 데이터 무결성 훼손을 방지합니다.

4. **외부 API 키 노출 차단 (Proxy Pattern 적용)**: 프론트엔드에서 알라딘 도서 검색 API를 직접 호출할 경우 `TTBKey`가 브라우저에 노출되는 치명적인 보안 결함이 발생합니다. 반드시 백엔드 서버를 거쳐(Proxy) 외부 API와 통신하도록 아키텍처를 강제합니다.

5. **Refresh Token Rotation (재발급 보안)**: `/auth/refresh` 호출 시 Access Token뿐 아니라 Refresh Token도 동시에 교체 발급합니다. 이전 Refresh Token은 즉시 무효화되며, 이미 사용된 토큰으로 재발급 시도 시 해당 사용자의 모든 세션을 강제 만료합니다. 현재 토큰 TTL: Access Token 1800초(30분), Refresh Token 604800초(7일).

6. **최초 관리자(Admin) 계정 생성**: 일반 회원가입으로는 `ROLE_ADMIN` 권한을 얻을 수 없습니다. 초기 관리자 계정은 Flyway DB 마이그레이션 스크립트(`db/migration/`)를 통한 시딩(Seeding) 또는 개발 환경 전용 부트스트랩 API로 생성하며, 운영 환경에서 해당 경로는 반드시 비활성화해야 합니다.

### ⚡ 성능 (Performance) 및 예외 제어, 동시성

1. **도서 대출/예약 락 (Lock) 처리 (동시성 방어)**: 이벤트 기간이나 신간 입고 시 잔여 수량이 1권인 인기 서적에 다수의 트랜잭션이 수 밀리초 차이로 발생한다면 DB가 여러 명에게 모두 빌려준 것으로 오인할 수 있습니다. **Pessimistic DB Lock (비관적 락)** 이나 **Optimistic Lock (@Version)** 방식의 로직 방어를 경유해 1인에게만 도서를 안전하게 할당하고 나머지는 대출 실패 처리를 내려합니다.

2. **비동기 메시지/알림 응답 속도 보완**: 도서 예약의 자동 통보 및 반납 독촉 진행 시 카카오톡이나 이메일 등의 외부 API 요청으로 서비스 전체 로딩이 병목 현상에 빠져 UX가 손상되지 않도록 대응해야 합니다. 발송 로직 자체는 메인 스레드를 차지하지 않는 **Spring ApplicationEventPublisher** 또는 **스레드 풀(@Async)**을 활용하여 핵심 로직에서 분리하는 구조가 권장됩니다.

3. **실시간 알림 (Redis Pub/Sub + SSE)**: 대출 승인, 반납 완료, 예약 순번 도래, 희망도서 승인/반려 등의 이벤트 발생 시 Redis Pub/Sub을 통해 메시지를 발행하고, Spring SSE(Server-Sent Events)를 통해 클라이언트에게 실시간으로 전달합니다. 알림 히스토리는 DB에 저장되며, 읽음/미읽음 상태를 관리합니다.

## 6. 상세 기능 요구사항

### ✅ 우선순위 정의

- **R1 (필수, Must Have)**: 시스템 운영을 위해 반드시 구현되어야 하는 핵심 기능. (예: 로그인, 도서 신청/승인, 대출/반납, 기본 알림 등)
- **R2 (선택, Should Have)**: 필수는 아니지만 사용자 편의와 관리 효율성을 높이는 기능. (예: 연체 관리, 도서 정렬/필터, 기본 통계, 인기 도서 추천 등)
- **R3 (확장, Nice to Have)**: 서비스 고도화를 위한 부가 기능. 초기 개발 단계에서는 제외 가능. (예: 개인화 추천, 상세 이력 추적, 공지사항, 고급 통계 등)

### 👤 사용자(User) 기능

#### 1. 회원 관리
- **[R1]** 회원가입 / 로그인 (카카오 OAuth 및 이메일 가입 지원)
- **[R1]** 프로필 관리 (이름, 부서, 이메일, 프로필 사진)
- **[R1]** 권한: 기본적으로 User 권한 부여

#### 2. 희망도서 신청
- **[R1]** 도서 신청 (도서명, 저자, 출판사, ISBN, 신청 사유 입력)
- **[R1]** 신청 내역 조회 (승인 대기 / 승인됨 / 거절됨)
- **[R2]** 중복 신청 방지 (보유 도서와 중복 시 알림)

#### 3. 도서 검색 & 대출
- **[R1]** 도서 검색 (제목, 저자, 출판사, ISBN)
- **[R1]** 도서 상세 정보 조회 (제목, 저자, 출판사, ISBN, 상태)
- **[R1]** 대출/반납 (대출 가능 권수 제한, 연체 시 대출 제한)
    - **대출 정책**: 1회 최대 10권, 대출 기간 14일
    - **반납 정책**: 반납 예정일까지 도서 반납 필수
    - **연체 처리**: 연체 시, 연체일수만큼 대출 정지
- **[R2]** 반납 연장 요청 (관리자 승인 필요, 해당 도서 예약자 없는 경우에 한해 1회 7일 연장 가능)
- **[R2]** 재대출 정책
    - 동일 도서 재대출은 반납 후 7일 이후 가능
    - 단, 예약자가 없을 경우 1회에 한해 즉시 재대출 가능
    - 재대출 횟수 제한: 동일 도서 연속 재대출은 최대 1회
- **[R2]** 필터/정렬 (대출 가능 여부, 인기순, 최신 등록순, 가나다순)

#### 4. 대출 예약
- **[R1]** 예약 대상: 현재 대출 중인 도서만 예약 가능 (※ 대출 가능/소장 중 도서는 예약 불가)
- **[R1]** 예약 방법: 도서 상세 화면에서 "대출예약신청" 버튼 클릭
- **[R1]** 예약 알림: 예약된 도서가 반납되면 예약자에게 실시간 알림 발송
- **[R1]** 예약 보관: 반납된 예약 도서는 4일간 별도 보관 후 자동 취소 처리
- **[R2]** 예약 제한
    - 1인당 최대 2권 예약 가능
    - 도서 1권당 최대 예약자는 2명까지
    - 예약자가 2명 이상인 경우, 이후 예약 불가
    - 예약 2순위 이용자는 선순위 취소/만료 시 자동 승계

#### 5. 마이페이지
- **[R1]** 나의 대출 현황 (대출 도서 리스트, 반납 예정일)
- **[R1]** 나의 신청 현황
- **[R2]** 나의 대출/반납 이력
- **[R2]** 나의 예약 현황 (예약 중 도서, 예약 순위, 만료일)
- **[R1]** 인앱 알림함 (읽음/미읽음 상태 관리)

#### 6. 알림 시스템
- **[R1]** 신청 승인/거절 알림
- **[R1]** 반납 예정일 알림 (실시간)
- **[R1]** 예약 도서 도착 알림 (실시간)
- **[R2]** 연체 안내 알림

### 🛠 관리자(Admin) 기능

#### 1. 도서 관리
- **[R1]** 신규 도서 등록 (구매 완료 후 반영)
- **[R1]** 도서 상태 관리 (대출 가능/대출 중/분실/폐기)
- **[R2]** 도서 정보 수정/삭제

#### 2. 대출 관리
- **[R1]** 현재 대출자 확인
- **[R1]** 반납 처리 (반납 시 도서 상태를 대출 가능으로 변경. 연체 여부 확인 및 대출 제한 적용)
- **[R2]** 연체 사용자 관리 (알림 전송, 연체일수만큼 대출 제한 적용)
- **[R2]** 반납 연장 승인 (예약이 없는 경우 1회 7일 연장 승인 가능)
- **[R2]** 재대출 관리 (동일 도서 재대출 요청 확인, 예약 여부 및 횟수 제한 검증)

#### 3. 예약 관리
- **[R1]** 예약 목록 조회 (예약 도서, 예약자 정보, 순위 확인)
- **[R1]** 예약 도착 알림 전송 (자동)
- **[R1]** 예약 보관 기간 관리 (4일 경과 시 자동 취소 처리)
- **[R2]** 예약 제한 관리 (1인 최대 예약 권수 확인, 1권당 최대 예약자 수 검증)
- **[R2]** 예약 자동 승계 처리 (선순위 취소/만료 시 후순위자에게 알림 발송)

#### 4. 신청 관리
- **[R1]** 희망도서 신청 목록 확인
- **[R1]** 신청 승인/거절 처리
- **[R2]** 승인 시 구매 상태 → 보유 완료 처리

#### 5. 회원 관리
- **[R2]** 회원 리스트 조회
- **[R2]** 권한 변경 (User ↔ Admin)
- **[R3]** 회원 정보 수정/비활성화

#### 6. 통계/리포트
- **[R2]** 월별 대출량 통계
- **[R2]** 인기 도서 순위
- **[R3]** 희망도서 신청 현황
- **[R3]** 사용자별 이용 통계
